10 Web Accessibility Tips

by Charlie Thomason

1. Skip To Content

A "Skip to Content" or "Skip Navigation" link should be the first element in the document. It should be hidden except to screen readers and appear only on focus.

There are 2 ways to implement this link: set the href value to the ID of whatever element immediately follows the navigation, OR set a JS click event to set .focus() on an element with tabindex.

Example

Skip to Content
This is the main content.
<a href="#main-content" id="skip-content">Skip to Content</a>
<div id="main-content">This is the main content.</div>

<style>
  #skip-content {
    position:absolute;
    left:-9999px;
    overflow:hidden;
  }
  #skip-content:focus {
    left:0;
    top:0;
  }
</style>

2. Abbreviations and Foreign Languages

Use the tag for abbreviations and tag for foreign language. Set the "title" attribute equal to whatever the abbreviation or foreign expression means.

Example

James has 1 lb. of beef from NC. King Henry VIII said it had a certain je ne sais quoi.

<p>James has 1 <abbr title="pound">lb.</abbr> of beef from <abbr title="North Carolina">NC</abbr>. King Henry <abbr title="the eighth">VIII</abbr> said it had a certain <dfn title="French: indefinable quality">je ne sais quoi</dfn>.</p>

3. Roles

If you only use 1 role attribute on your whole page, it should be "navigation" for the main nav. It's extremely useful for screen reader users to know that a group of links is the main navigation and not part of the page content.

Example

  • Home
  • About
  • Shop
<ul role="navigation"> 
  <li><a href="#">Home</a></li> 
  <li><a href="#">About</a></li> 
  <li><a href="#">Shop</a></li> 
</ul>

4. Setting Focus To Non-Focusable Elements

You cannot use .focus() on a <div> that doesn't have tabindex. Setting tabindex="0" makes the element always tab-focusable; setting tabindex="-1" makes the element only tab-focusable by using .focus().

You can always set focus to an element by clicking a link whose href is set to the target element's ID (e.g.: "Skip to Content" links). Links and form elements are the only things that will always receive tab focus out-of-the-box. For tooltips, clicking the tooltip icon should set .focus() to the tooltip popup, and closing the popup should return focus to the icon.

Example

more info
This tooltip is a div with a tabindex value so that you can focus on it with JavaScript.
<label for="form-5" class="tooltip-label">Phone number</label><a href="#" class="tooltip-icon">more info</a><br> 
<input type="tel" id="form-5" class="input-with-tooltip"> 
<div class="tooltip-help" tabindex="-1" role="tooltip">This tooltip is a div with a tabindex value so that you can focus on it with JavaScript.</div>

<script>
$('.tooltip-icon').click(function(e) {
  e.preventDefault();
  var $target = $(this).siblings('.tooltip-help');
  $target.toggle();
  $target.focus();
});

$('.tooltip-help').click(function(e) {
  $(this).toggle();
  $(this).siblings('.tooltip-icon').focus();
});
</script>

5. Accessible Alternatives to Form Labels

Instead of using a <label> with a "for" attribute, you can use any HTML tag as a label and set "aria-labelledby" to the element's ID. Alternatively, you can set a "title" or "aria-label" attribute on the input with a value of whatever the label text should be. These options can be especially useful for search inputs where a label is undesirable. However, there is other functionality provided by an actual <label> tag that these other solutions cannot provide.

Note: the placeholder attribute should never be used as a label.

Example

Search

uses aria-labelledby

uses a "title" attribute

uses aria-label 
<h3 id="form-label-3">Search</h3> 
<input type="text" aria-labelledby="form-label-3"> 
<input type="text" title="Search for stuff"> 
<input type="text" aria-label="Search for more stuff">

6. aria-describedby

If you have help text below a form element you should set a aria-describedby attribute on the corresponding form element. The value of the aria-describedby should be equal to the ID of the help text tag.

Example

Password must be 8-16 characters and contain only letters, numbers, and special characters.

<label for="form-4">Password</label>
<input type="password" id="form-4" aria-describedby="form-help-4"> 
<p id="form-help-4">Password must be 8-16 characters and contain only letters, numbers, and special characters.</p>

7. Fieldset and Legend

When you have a group of checkboxes or radio buttons, they should be grouped inside a <fieldset> with a <legend> tag as the group label. Alternatively you can use aria-labelledby on each checkbox/radio. Otherwise, screen readers will simply read the text of each checkbox/radio label without any context for what that input represents.

Example

Pizza toppings 

<fieldset>
  <legend>Pizza toppings</legend>
  <input id="pepperoni" type="checkbox" name="toppings" value="pepperoni" checked="checked">
  <label for="pepperoni">Pepperoni</label><br>
  <input id="sausage" type="checkbox" name="toppings" value="sausage">
  <label for="sausage">Sausage</label><br>
  <input id="bacon" type="checkbox" name="toppings" value="bacon">
  <label for="bacon">Bacon</label>
</fieldset>

8. Knowing When ARIA Tags Are Necessary

If an element has a disabled, checked, selected, or required attribute, it does not also need an aria-disabled, aria-checked, aria-selected, or aria-required attribute. These are only for custom interactive elements that do not provide this functionality by default.

Example

<label for="form-7">Unnecessary aria-disabled attribute</label> 
<input type="text" id="form-7" disabled="disabled" value="Chicken" aria-disabled="true">

Example

On Off 
<span class="on-off-toggle on" aria-checked="true">On</span>
<span class="on-off-toggle" aria-checked="false">Off</span>

<script>
  $('.on-off-toggle').click(function() {
    if (!$(this).hasClass('on')) {
      $('.on-off-toggle.on').removeClass('on').attr('aria-checked','false');
      $(this).addClass('on').attr('aria-checked','true');
    }
  });
</script>

9. Table Tips

  1. thead, tbody, and tfoot provide no accessibility or semantic value whatsoever.
  2. <th> tags can and should be used for both column AND row headings.
  3. The scope attribute should be used any time you use a <th>, and should have a value equal to "column" or "row."
  4. Tables should always have a <caption> tag as the very first element, even if it's hidden.

Example

Team Players
Names Age Score Position
Charlie 31 200 Goalie
Sam 35 150 Sweeper
David 28 225 Forward 
<table>
  <caption>Team Players</caption> 
  <thead> 
    <tr> 
      <th scope="col">Names</th> 
      <th scope="col">Age</th> 
      <th scope="col">Score</th> 
      <th scope="col">Position</th> 
    </tr> 
  </thead> 
  <tbody>
    <tr> 
      <th scope="row">Charlie</th> 
      <td>31</td> 
      <td>200</td> 
      <td>Goalie</td> 
    </tr> 
    <tr> 
      <th scope="row">Sam</th> 
      <td>35</td> 
      <td>150</td> 
      <td>Sweeper</td> 
    </tr> 
    <tr> 
      <th scope="row">David</th> 
      <td>28</td> 
      <td>225</td> 
      <td>Forward</td> 
    </tr> 
  </tbody> 
</table>

10. aria-live

An ARIA live region is a mechanism for notifying screen readers when content is updated on the page. Screen reader support is currently very inconsistent. It has 3 possible values: off, polite, and assertive. Assertive will interrupt whatever the screen reader is currently saying and has the best browser support.

Update 

<div id="result1" aria-live="assertive"></div> 
<a href="#" class="btn btn-success" id="update1">Update</a>

<script>
  $('#update1').click(function(e) {
    e.preventDefault();
    $('#result1').html('Updated!');
  });
</script>
